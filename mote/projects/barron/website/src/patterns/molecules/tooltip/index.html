Tooltip

The Tooltip Molecule is slightly different from other molecules in that its DOM nodes are manipulated by JS. the Block level class Tooltip is decorated onto the Node after it has been applied to its target (see below). We only make use of the Tooltip-contents class when we are writing our HTML. This is used in conjuction with Element classes on the tooltips close button Tooltip-closer and Tooltip-title for its title. The rest is added by the tooltip-wrapper() function 
            
                
{% render_element element.aspect.molecules.tooltip %}
The tooltips are generated by a tooltip-wrapper() that extends drop.js, which has a dependacy upon tether.io (these libraries are included by the wrapper). The tooltip-wrapper() watches for html elements with the class Tooltip-contents and then uses the values from the data attributes: data-tooltip-target and data-tooltip-config. The tooltipTarget is the node on which the tooltip will attach. The tooltipConfig is an identifier for the js config object sitting on the window.Barron object.
window.Barron is defined in the <head>.
The example shown in templates/home is a mock that can be used as the base for the tooltip that displays over the mashead. 
            
                
Barron.tooltip1 = {
    'breakpoint': 1024,

    'desktopOptions': {
        'classes': 'Tooltip u-typography--inverted ',
        'position': 'right middle',
        'openOn': 'click',
        'tetherOptions': {
            'constraints': []
        }
    },

    'mobileOptions': {
        'classes': 'Tooltip u-typography--inverted ',
        'position': 'top center',
        'openOn': 'click',
        'tetherOptions': {
            'constraints': []
        }
    }
}
